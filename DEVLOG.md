# tekken8_auto_tracker

## 概要

`tekken8_auto_tracker` は、鉄拳8の勝敗画面を監視し、WIN / LOSE を自動検出して戦績記録へつなげることを目的とした検証用プロジェクトです。

現在は、Python・OpenCV・MSS を使用して画面キャプチャと勝敗検出の検証を行っています。

## 現在の進捗

### 完了済み

* Python環境構築
* venv構築
* requirements.txt導入
* OpenCV導入
* MSS導入
* スクリーンショット取得

* 監視間隔を1秒から0.2秒へ変更
* ROIによる判定範囲限定を検証

## 現在の動作

* 鉄拳8の画面を一定間隔でキャプチャする
* WIN / LOSE テンプレートと画面を比較する
* WIN検出時に勝利数を加算する
* LOSE検出時に敗北数を加算する
* 同じ結果を連続カウントしない
* `Ctrl + C` で監視を停止できる

## 検証結果

### 成功したこと

* スクリーンショット取得に成功
* テンプレート画像単体では `score = 1.000` を確認
* WIN / LOSE の検出自体は成功
* `sleep = 0.2` に変更することで取りこぼしは一部改善
* リアルタイム監視と戦績カウントの基本動作を確認

### 発生した問題

* 明るいステージや砂浜系ステージで取りこぼしが発生
* 試合中の画面でも `WIN score` / `LOSE score` が 0.50～0.60 程度出る
* 実際の勝敗画面でもスコアが 0.50～0.60 程度に留まる場合がある
* WIN / LOSE のスコア差が小さく、閾値調整だけでは安定しない
* ゴーストバトルや一部UI文字列で誤検出の可能性あり
* ROI導入による大幅な改善は確認できなかった

## テンプレートマッチング検証

### 実施内容

* OpenCV の `cv2.matchTemplate()` を使用
* WIN / LOSE のテンプレート画像を作成
* グレースケール化して比較
* テンプレート画像を文字ギリギリまでトリミング
* ROIで画面中央付近に探索範囲を限定

### 結果

* テンプレート元画像では高スコアを確認
* 実ゲーム画面ではステージ背景や明度の影響を強く受ける
* 試合中と勝敗画面で十分なスコア差が得られなかった
* 閾値を下げると誤検出が増え、上げると取りこぼしが発生する
* テンプレートマッチング単独での安定運用は難しい可能性が高い

## 現時点の判断

勝敗検出の仕組み自体は成立しているものの、テンプレートマッチング方式は鉄拳8の背景演出・ステージ明度・画面エフェクトの影響を受けやすい。

そのため、今後はテンプレートマッチングを完成版の主軸にするのではなく、OCRによる文字認識方式を次の候補として検証する。
